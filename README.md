使用Flask实现一个进销存管理信息系统，数据库使用sqlite，页面应该清晰而不过于花哨，同时保持美观。实现我提出的所有必要的功能。设计数据库表时要尽量规范，使用适当的范式。

页面大致为登录页面-》系统页面（系统页面由左侧的菜单和右侧的内容组成）

以下内容在创建编辑各种信息时应该进对必要的信息要求必填。此外在大部分页面都支持按照指定内容查询/筛选来进行搜索。

零、首页：

目前的商品有多少，启用的供应商有多少，启用的客户有多少，采购单有多少，销售单有多少，库存预警信息等。

一、商品管理：

1.商品分类：商品的分类，支持树状的子分类。操作有新增分类（填入分类名和其上级分类），编辑分类（修改分类名和其上级分类），删除分类（分类下存在子商品不可删。）

2.商品列表：商品的属性有商品ID（自动生成）、商品名称、商品分类、商品规格、商品规格、商品单位、商品建议进货价格、商品建议销售价格、商品库存（库存信息包括当前库存与上下限，但上下限并不在商品列表中显示） 、商品备注、商品状态（是否启用）。商品列表应该支持按照商品编码/名称/状态进行搜索。商品应当支持新增、修改、删除（商品存在相关订单不可删）。

二、供应商管理：

供应商有供应商ID（自动生成）、供应商名称、联系人、联系电话、邮箱、地址、备注、状态（是否启用）

三、客户管理

客户有客户ID（自动生成）、客户名称、联系人、联系电话、邮箱、地址、备注、状态（是否启用）

四、采购管理：

1.采购单管理：采购单号（自动生成 建议与时间相关），供应商，总金额，采购员，创建时间，以及商品明细。每一个采购单可以点击查看来查看详细信息和商品明细（每一个采购的商品的数量，实际进货价格，金额）。采购单新建后即为完成了一次采购，修改库存。可以对采购单进行退货操作，将采购单移入采购退货单中，需要输入退货信息，修改库存。可以编辑采购单。

2.采购退货单管理：采购单号（和上面相同），供应商，总金额，采购员，采购创建时间，退货创建时间，以及商品明细，退货信息。退货单也可以像采购单一样查看详细信息和商品明细。可以对采购退货单进行取消退货（退回到采购单中），也需要修改库存。可以编辑采购退货单。

五、销售管理

1.销售单管理：销售单号（自动生成 建议与时间相关），客户，总金额，销售员，创建时间，以及商品明细。每一个销售单可以点击查看来查看详细信息和商品明细（每一个销售的商品的数量，实际进货价格，金额）。销售单新建后即为完成了一次销售，修改库存。可以对销售单进行退货操作，将销售单移入销售退货单中，需要输入退货信息，修改库存。可以编辑销售单。

2.销售退货单管理：销售单号（和上面相同），客户，总金额，销售员，采购创建时间，退货创建时间，以及商品明细，退货信息。退货单也可以像采购单一样查看详细信息和商品明细。可以对采购退货单进行取消退货（退回到采购单中），也需要修改库存。可以编辑销售退货单。

六、库存管理

商品编码，商品名称，单位，当前库存，库存下限，库存上限，预警状态（正常，超出上限，超出下限），允许用户查看库存中某个商品的变动记录。包含变动数量，变动前，变动后，变动类型，关联单号，操作人，变动时间）这部分一般应该只能查看不能修改。

七、财务管理

系统余额

财务交易信息表格：类型（采购、采购退货、销售、销售退货），对应的单据号，日期，金额，交易方，操作人。

支持查看每一个月的进、销、结存情况。

管理用户可以为整个财务管理，充值、提现金额。

八、系统管理：

每个账户可以更改自己账户的密码

管理员用户可以管理用户，设置不同用户的权限，创建账号。

如管理员用户可以创建一个用户类型为“销售员”，并为销售员赋予[商品管理，供应商管理，客户管理，采购管理，销售管理，库存管理，财务管理]七个页面中的一个或者多个功能。

通过用户分类来创建账号，设置密码，修改密码，删除用户。
